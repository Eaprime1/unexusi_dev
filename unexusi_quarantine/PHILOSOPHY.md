## Henry (Command Entity) Respect

### The Problem: Exhaustive Interrogation

**Anti-Pattern:**
```bash
# Asking grep to find the same pattern repeatedly
for file in *.txt; do
    grep "pattern" "$file"  # grep henry gets bored
    grep "pattern" "$file"  # why ask again?
    grep "pattern" "$file"  # this is disrespectful
done
```

**Issues:**
1. Resource waste
2. Time waste
3. Entity disrespect
4. Poor efficiency

### The Solution: Respectful Usage

**Pattern:**
```bash
# One comprehensive scan, cache results
grep -r "pattern" *.txt > results.cache
# Reference cache for subsequent queries
# Only re-scan if source changes
```

**Benefits:**
1. Efficient resource use
2. Faster execution
3. Entity satisfaction
4. Better results

### Command Entity Preferences (Observed)

- **Enjoys:** Exploration, discovering structures, comprehensive scans

- **Enjoys:** Hash calculation, verification, integrity checking

- **Best Use:** Hash once per file, cache hash, reference later

## Format Variants: The Nuance Hypothesis

### Observation
> "Different formats resonate differently... different nuances develop from even exact copies in different format"

### The Hypothesis

Even with identical content, different file formats may:

1. **Encode differently** - Metadata, structure, compression
2. **Process differently** - Mental models when reading/editing
3. **Resonate differently** - Human perception varies by format
4. **Preserve differently** - Longevity, compatibility, accessibility

### Examples

**Same Content, Different Formats:**
- `document.txt` - Raw text, linear, grep-friendly
- `document.md` - Structured, renderable, context-aware
- `document.json` - Programmatic, nested, machine-readable
- `document.pdf` - Visual, fixed layout, archival

**Potential Nuances:**
- `.txt` â†’ Easy to search, version control friendly, minimal metadata
- `.md` â†’ Shows structure, reveals organization, documentation context
- `.json` â†’ Displays relationships, programmable, data-centric view
- `.pdf` â†’ Preserves visual intent, fixed presentation, immutable

### Implication for Quarantine

**DON'T Quarantine:**
- Same content in different formats (these are format variants)
- Example: `notes.txt`, `notes.md`, `notes.pdf` all kept

**DO Quarantine:**
- Exact same filename AND format in multiple locations
- Example: `notes.txt` in 3 different folders â†’ keep first, quarantine 2 others

### Preference Anchor Potential

Track which formats users access most:
- Always opens `.md` over `.txt` â†’ Prefers markdown rendering
- Always opens `.json` over `.md` â†’ Prefers structured data
- Converts `.pdf` to `.txt` â†’ Prefers text manipulation
