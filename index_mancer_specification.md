### SQLite Schema (for relationship queries)
```sql
CREATE TABLE zero_points (
    id TEXT PRIMARY KEY,
    date DATE,
    file TEXT,
    type TEXT,
    charge_level INTEGER
);

CREATE TABLE threads (
    id TEXT PRIMARY KEY,
    name TEXT,
    status TEXT,
    started TEXT,
    charge_accumulated INTEGER
);

CREATE TABLE concepts (
    id TEXT PRIMARY KEY,
    name TEXT,
    introduced TEXT,
    status TEXT
);

CREATE TABLE relationships (
    id INTEGER PRIMARY KEY,
    source_id TEXT,
    target_id TEXT,
    relationship_type TEXT,
    strength TEXT,
    FOREIGN KEY (source_id) REFERENCES zero_points(id),
    FOREIGN KEY (target_id) REFERENCES zero_points(id)
);

-- Enables queries like:
-- "Show all zero points introducing prime mechanics concepts"
-- "Find all threads still active after 30 days"
-- "Map relationship clusters by strength"
```

## Usage Workflow

### Initial Setup
```bash
# First time use
python index_mancer.py --init
# Creates index.json, scans existing zero points
# Generates initial zero_point_index.md
```

### Regular Operation
```bash
# After new conversation archived
python index_mancer.py --scan
# Detects new ZP file, extracts metadata
# Updates index.json and zero_point_index.md
# Reports new relationships/concepts found

# Search operations
python index_mancer.py --search thread "Archive Management"
python index_mancer.py --search date "2025-11"
python index_mancer.py --search concept "prime mechanics"
```

```bash
# When feeling overwhelmed
```
