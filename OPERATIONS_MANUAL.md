## Emergency Protocols

### When Overwhelmed

**Immediate actions**:
1. **STOP** (full pause_engaged)
2. Close all tools (reduce input)
3. Find environmental tether (cool air, physical anchor)
4. Breathe (literal br[e]ath[e])
5. Don't force recovery (let it come naturally)

**Recovery process**:
1. Return to last stable checkpoint
2. Document what triggered overwhelm
3. Design prevention for next time
4. Resume at slower pace
5. Celebrate resumption (courage recognized)

### When Tools Fail

**Remember**:
- Tools enable, don't replace human capability
- Manual process always available
- Tool failure â‰  framework failure
- Operational proof > automated perfection

**Response**:
1. Fallback to manual process
2. Document what tool couldn't handle
3. Improve tool specification
4. Test with simpler case
5. Rebuild confidence through success

### When Patterns Won't Emerge

**Strategic ignoring isn't working**:
1. Maybe pattern isn't there (that's okay)
2. Maybe needs more time (background burner needs days, not minutes)
3. Maybe needs different perspective (ask different question)

**Trust**:
- Not everything resolves
- Undefinable constant exists
- Mystery is feature, not bug
- Operational proof emerges when ready
